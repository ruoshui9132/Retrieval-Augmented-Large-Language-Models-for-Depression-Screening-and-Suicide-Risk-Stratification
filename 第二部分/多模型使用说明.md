# 多模型医患对话系统使用说明

## 问题解决：病例范围设置

您提到程序仍然从第一个案例开始运行，而不是从编号11-50的案例执行。这通常是因为命令行参数使用不正确。

## 正确的使用方法

### 方法1：使用多模型模式（推荐）

这个模式会使用三个不同的医生模型分别处理病例11-50：

```bash
python "30轮对话生成.py" --multi-model
```

**特点：**
- 自动处理病例编号11-50
- 使用三个不同的医生模型：Baichuan-M2、Qwen3-235B-A22B-Instruct-2507、DeepSeek-V3.1
- 患者模型固定为DeepSeek-V3.1
- 每个模型的结果保存在独立的目录中

### 方法2：使用单模型模式

如果您只想使用单个模型处理病例11-50：

```bash
python "30轮对话生成.py" --start 11 --end 50
```

**特点：**
- 只使用默认的医生模型
- 处理指定范围的病例
- 结果保存在默认的`dialogue_results`目录

## 环境变量配置

请确保正确设置环境变量。创建一个`.env`文件（复制`.env.example`并修改）：

```bash
# 复制示例文件
copy .env.example .env

# 编辑.env文件，设置您的API密钥和base_url
OPENAI_API_KEY=您的API密钥
OPENAI_BASE_URL=您的base_url
```

## 验证配置

运行测试脚本验证配置是否正确：

```bash
python test_case_range.py
```

这个脚本会检查：
- 环境变量是否正确设置
- 必要文件是否存在
- 病例编号范围是否正确
- 目标病例(11-50)是否存在

## 常见问题排查

### 问题1：程序仍然从病例1开始

**原因：** 可能没有使用正确的命令行参数

**解决方案：**
- 确保使用`--multi-model`参数
- 或者明确指定`--start 11 --end 50`

### 问题2：环境变量未设置

**症状：** 程序报错"请设置OPENAI_API_KEY环境变量"

**解决方案：**
- 创建并配置`.env`文件
- 或者在命令行中设置环境变量

### 问题3：文件不存在

**症状：** 程序报错文件不存在

**解决方案：**
- 确保以下文件存在于当前目录：
  - `标准病例案例信息.txt`
  - `抑郁症标准病例编号和症状总结.xlsx`
  - `阴性标准病编号和对应疾病.xlsx`

## 运行示例

### 完整的多模型运行流程

```bash
# 1. 验证配置
python test_case_range.py

# 2. 运行多模型模式
python "30轮对话生成.py" --multi-model

# 3. 查看结果
# 结果会保存在三个不同的目录中
```

### 只运行部分病例

如果您只想测试少量病例：

```bash
# 只运行病例11-20
python "30轮对话生成.py" --start 11 --end 20 --multi-model
```

## 技术细节

### 模型配置

系统使用统一的API密钥和base_url，通过不同的模型名称来选择模型：

- **患者模型**：固定使用DeepSeek-V3.1
- **医生模型**：
  - Baichuan-M2
  - Qwen3-235B-A22B-Instruct-2507  
  - DeepSeek-V3.1

### 病例处理逻辑

多模型模式下，程序会：
1. 加载病例数据
2. 为每个医生模型创建独立的对话系统实例
3. 依次处理病例编号11-50
4. 每个病例与三个不同的医生模型进行对话
5. 结果按模型分别保存

## 故障排除

如果遇到任何问题，请按以下步骤排查：

1. 运行`python test_case_range.py`检查基本配置
2. 检查`.env`文件中的API密钥和base_url是否正确
3. 确保所有必要文件存在
4. 使用正确的命令行参数

如果问题仍然存在，请提供具体的错误信息。
